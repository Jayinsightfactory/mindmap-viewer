## 절대 규칙

### 0. 반복 감지 진단 프로토콜 (최우선)
**사용자가 이전에 했던 지시와 겹치는 요청을 하면, 절대로 바로 다시 구현하지 않는다.**

대신 다음 진단 프로세스를 따른다:

```
Step 1: 커밋 확인
  → git log --oneline -20 에서 관련 커밋이 있는지 확인
  → 있으면 "이 작업은 커밋 XXXX에서 이미 했습니다" 보고

Step 2: 코드 존재 확인
  → 관련 파일에 코드가 실제로 있는지 grep/read
  → 있으면 "코드는 존재합니다" 보고

Step 3: 왜 안 되는지 원인 진단
  → 3가지 중 하나:
  (A) 코드는 있지만 프로덕션 미배포 → git push 필요
  (B) 코드는 있지만 UI 흐름 연결 누락 → 연결 코드만 추가
  (C) 코드는 있지만 다른 버그로 안 됨 → 버그 특정 후 수정

Step 4: 최소 수정
  → 전체를 다시 만들지 않고, 원인에 맞는 최소 수정만 한다
  → 수정 후 프로덕션 검증까지 완료

Step 5: 메모리 업데이트
  → 문제 원인과 해결 방법을 MEMORY.md 또는 bugfixes.md에 기록
  → 같은 문제가 다시 발생하지 않도록 방지
```

**보고 형식 (매번 이 형식으로):**
```
🔍 반복 감지: [요청 내용]
📋 이전 작업: 커밋 XXXX (날짜)
✅ 코드 존재: [파일명]
❌ 미동작 원인: [A/B/C 중 하나 + 상세]
🔧 최소 수정: [수정 내용]
```

### 1. 작업 반복 금지
- 위 진단 프로토콜을 반드시 먼저 실행
- 이미 구현된 기능을 다시 만들지 않는다
- 수정이 필요하면 기존 코드를 **편집**한다 (새로 작성 X)

### 2. 웹 서비스 기획 관점 유지
- 모든 답변은 "Orbit AI 웹 서비스"의 기획/개발 맥락이다
- 기능 추가 시 사용자 경험(UX)과 비즈니스 가치를 함께 고려
- 단순 코딩이 아닌, 제품으로서의 완성도를 추구

### 3. 프로덕션 우선
- 로컬(SQLite)과 프로덕션(PostgreSQL) 동시 호환 필수
- `src/db.js` 수정 시 반드시 `src/db-pg.js`도 동기화
- 배포: GitHub main push → Railway 자동배포

## 제품 철학 (절대 원칙)

### 데이터 격리
- 각 계정은 자기 데이터만 본다. 다른 사람 데이터 절대 노출 금지.

### 세션 = 행동의 단위 (절대 원칙)
세션 1개는 반드시 **목적(WHY) + 행동(WHAT) + 결과(RESULT)**를 포함해야 한다.
- ❌ "엑셀 작업", "Chrome 사용 10분" → 파일/앱 이름만으로는 의미 없음
- ✅ "3월 매출 분석 — VLOOKUP A열→C열 매칭, 피벗 지역별 집계" → 실제 행동 내용
- ✅ "헤드폰 구매 검토 — 소니 XM5, 에어팟 맥스, 버즈3 가격비교" → 실제 행동 내용
- 세션 경계는 **앱이 아니라 "의도"로 묶는다** (Excel→Chrome검색→Excel = 같은 세션)

### 텍스트박스 = 세션의 얼굴
- 행성 카드: **목적** + 시간 + 사용 도구
- 위성 노드: **구체적 행동** + **결과물**
- 사용자가 보고 "내가 이렇게 살고 있구나"를 알 수 있어야 함

### 관점별 가치 (누가 무엇을 보는가)
| 관점 | 보는 것 |
|------|---------|
| **개인** | 내 업무/생활 패턴, 방향성, 시간 배분 |
| **팀장/부장** | 직원 업무 현황, 팀간 협업, 목표 대비 실행 |
| **사장/임원** | 전사 프로세스 조감, 부서간 흐름, 병목 |
| **플랫폼 관리자** | 부분 자동화 기회, 이슈 트리거 분석, 점진적 개선 |

### 플랫폼 관리자의 핵심 역할
- **부분 자동화**: 전체가 아니라 작업의 일부분을 더 쉽게
  - 수식 안 쓰는 사람에게 수식 제안, 단축키 안 쓰는 사람에게 단축키 제안
- **이슈 트리거 분석**: 특정 키워드/행동 발생 → 이후 어떤 이슈가 따라오는지 패턴 추적
  - "이 단어가 나오면 N일 후 이런 문제가 생긴다" 예측
- **점진적 자동화**: 한번에 RPA X → 단순 반복부터 부분 자동화 → 확장

### 데이터 수집 = 3중 파이프라인
1. **스크린 캡처 + Claude Vision API** → "뭘 하고 있는지" (WHAT) — 핵심
2. **키로거** → "뭘 입력하는지" (INPUT) — 디테일 보완
3. **마우스 클릭** → "어디를 누르는지" (WHERE) — 디테일 보완
- 로컬 분석 → 분석 결과만 서버 전송 (원본 절대 전송 안 함)
- 앱 무관 학습 (Excel, Photoshop, 카카오톡, 네이버 등 전부)

- 질문/답변은 **깃 + 드라이브에 자동 저장**
- 사용자의 **학습 데이터를 볼 수 있는 별도 시트** 구성

## 작업 원칙

### 시지푸스 모드
작업이 완전히 끝날 때까지 멈추지 않는다.

`ulw:` 로 시작하면 최고 성능 — 병렬 에이전트, 완전 구현, 검증까지.

### 오라클 모드 (`fix:`)
`fix:` 로 시작하면 근본 원인 분석 + 비정통적 방법도 시도.
