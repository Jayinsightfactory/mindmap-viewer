 3. 타겟 고객 & 시장
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

■ 타겟 고객 유형:

  Type A — 소기업 (10~30명)
  · 현황: IT 시스템 거의 없음, 엑셀 100% 의존
  · 니즈: 기본 디지털화, 비용 절감
  · 접근: 2시간 부트캠프 → 즉시 효과 체험

  Type B — 중기업 (30~100명) ★ 주력 타겟
  · 현황: ERP/CRM 부분 도입, 연동 안됨
  · 니즈: 프로세스 자동화, 데이터 통합
  · 접근: 전체 진단 → 월정액 리테이너

  Type C — 중견기업 (100~300명)
  · 현황: 시스템 다수 운영, 비효율 잔존
  · 니즈: 고도화, AI 도입, 의사결정 지원
  · 접근: 엔터프라이즈 커스터마이징

■ 한국 시장 데이터:
  · 중소기업 수: ~700만 개
  · 디지털 전환 정부 지원금: 연 수조원
  · 경쟁사: 주로 대기업용 솔루션 (고가, 복잡)

 4. 수익 모델 (3단계 퍼널)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

■ Stage 1: 부트캠프 (Entry)
  · 형태: 2시간 무료/저가 세션
  · 내용: 엑셀 데이터 업로드 → 즉시 자동 분석 → 문제점 리포트
  · 목표: 가치 체험 → 전체 진단으로 전환
  · API: POST /api/bootcamp/start
         POST /api/bootcamp/:id/upload
         POST /api/bootcamp/:id/analyze
         GET  /api/bootcamp/:id/report

■ Stage 2: 전체 진단 (Convert)
  · 형태: 유료 1회성 서비스
  · 내용: 트래커 설치 → 2주 데이터 수집 → 6영역 종합 진단
  · 산출물: 진단 리포트 + 자동화 기회 목록 + ROI 시뮬레이션
  · API: POST /api/diagnosis/:companyId/run
         GET  /api/diagnosis/:companyId/latest
         POST /api/diagnosis/:companyId/simulate

■ Stage 3: 월정액 리테이너 (Retain)
  · 형태: 월 구독 서비스
  · 내용: 실시간 모니터링 + 자동 진단(매일) + 이상 탐지 + 개선 제안
  · 산출물: 월간 리포트 + 실시간 대시보드 + 컨설팅 동행
  · 자동화: company-crawler.js가 매일 자동 실행

■ 수익 구조:
  ┌──────────────┬──────────┬──────────┬──────────┐
  │              │ 부트캠프 │ 전체진단 │ 리테이너 │
  ├──────────────┼──────────┼──────────┼──────────┤
  │ 가격         │ 무료~50만│ 200~500만│ 월100~300│
  │ 기간         │ 2시간    │ 2주      │ 월 단위  │
  │ 전환율(예상) │ -        │ 30~40%   │ 50~60%   │
  │ LTV          │ 0        │ 350만    │ 연1,200만│
  └──────────────┴──────────┴──────────┴──────────┘

 ■ 5개 핵심 엔티티 + 관계 모델

  엔티티 구조:
  ┌─────────────────────────────────────────────────────────┐
  │                    Company (회사)                       │
  │  · industry, employee_count, revenue_range              │

  ├────────────┬────────────────────────────────────────────┤
  │            │                                            │
  │  Department (부서)              Process (프로세스)       │
  │  · head_name, head_count        · frequency, steps      │
  │  · budget_monthly               · automation_potential   │
  │  · automation_score             · bottleneck_score       │
  │  · efficiency_score             · estimated_savings_krw  │
  │            │                                            │
  │  Employee (직원)                System (시스템)          │
  │  · position, role               · category, vendor      │
  │  · ai_readiness                 · monthly_cost          │
  │  · workload_score               · integration_level     │
  │  · tracker_token ★              · satisfaction           │
  │  · tracker_active               · api_available         │
  └────────────┴────────────────────────────────────────────┘

  관계 (Links):
  · belongs_to — 소속 관계
  · depends_on — 프로세스 의존성
  · collaborates — 부서 간 협업
  · uses_system — 시스템 사용 관계

 ■ 주요 함수:
  · ensureCompanyTables(db) — 11개 테이블 자동 생성
  · createCompany / getCompany / updateCompany / deleteCompany
  · createDepartment / listDepartments / updateDepartment
  · createEmployee / listEmployees / updateEmployee
  · createProcess / listProcesses / updateProcess
  · createSystem / listSystems
  · getEmployeeByToken(db, token) — 토큰 기반 직원 조회 (로그인 불필요)
  · insertActivities(db, activities) — 벌크 활동 데이터 저장
  · getActivityStats(db, companyId, since) — 활동 통계 집계
  · buildCompanyGraph(db, companyId) — 3D 시각화용 노드/엣지 생성
  · updateBenchmark(db, industry, type, metric, value) — 벤치마크 갱신

 5-3. 데이터 학습 엔진 (Learning Engine)

 ■ 6가지 학습 분석:

     · 부서별 활동 분포
     · 최고/최저 활동 시간대 탐지

  2. 병목 탐지 (detectBottlenecks)
     · 반복 작업 감지 (동일 앱 장시간 사용)
     · 컨텍스트 스위칭 과다 (하루 50회 이상)

  3. 자동화 기회 발견 (discoverAutomationOpportunities)

     · 이메일 반복 업무 → 자동 분류/응답
     · 데이터 입력 반복 → 자동화 대상
     · 예상 절감액(원) 자동 계산

  4. 이상 징후 탐지 (detectAnomalies)
     · 심야 작업 (23시~05시)
     · 주말 작업 패턴
     · 비업무 활동 (게임, 쇼핑, SNS)
     · 급격한 패턴 변화

  5. 협업 패턴 분석 (analyzeCollaboration)
     · 부서 간 공유 활동 빈도
     · 커뮤니케이션 도구 사용 현황
     · 사일로(격리) 부서 탐지
     · 협업 네트워크 시각화 데이터

  6. 프로세스 자동 매핑 (autoMapProcesses)
     · 앱 사용 패턴에서 업무 프로세스 추론
     · 프로세스 흐름도 자동 생성
     · 비효율 구간 표시

 ■ 종합 학습 리포트:
  · generateCompanyLearningReport(db, companyId)
  · 6가지 분석 결과를 하나의 리포트로 통합
  · 핵심 인사이트 요약
  · 개선 우선순위 제시

     - development: VSCode, IntelliJ
     - design: 포토샵, 피그마
     - etc: 기타
  · 브라우저 URL (브라우저 활성 시)
  · 타임스탬프 (초 단위)

 ■ 데이터 전송:
  · POST /api/tracker/activities (30초 간격, 벌크 전송)

  · Authorization: Bearer trk_XXXXXXX

 ■ Windows 작업 스케줄러 등록:
  · 작업명: OrbitTracker
  · 트리거: 로그온 시 자동 실행 (-AtLogOn)

  · 키보드 입력 내용은 수집하지 않음 (키 횟수만)

  · Google Drive REST API v3
  · Multipart Upload (메타데이터 + 파일 바이너리)

 ■ 백업 파일 형식:
  · orbit-backup-2026-03-04T12-00-00.db (SQLite 백업)

  · 시스템 목록
