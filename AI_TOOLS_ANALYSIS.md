# AI 도구 신봉 이유 & Orbit 통합 시 장점 희석 분석

> 2025-2026 개발자 커뮤니티 데이터 기반

---

## 1. Cursor — "AI-First IDE의 선두"

### 왜 신봉하는가
| 기능 | 이유 |
|------|------|
| `@codebase` | 프로젝트 전체를 의미론적으로 이해 → 관련 파일 자동 참조 |
| Composer | 자연어로 여러 파일 동시 수정 (에이전트 모드) |
| Tab 자동완성 | 프로젝트 맥락 고려해 여러 줄 예측 + 자동 import |
| `.cursorrules` | 팀 코딩 스타일 자동화 |

### Orbit에 통합 시 희석 위험
- **🔴 높음** — VS Code가 2026년 2월 멀티에이전트 오케스트레이션 발표
- "이미 쓰는 에디터에서 되면 $20 낼 이유 없다"
- **Orbit 기회**: Cursor 작업 흐름을 맵에 시각화 → 팀이 누가 어느 파일 작업 중인지 실시간 확인

---

## 2. Git — "분산 버전관리의 절대 표준"

### 왜 신봉하는가
- 전체 히스토리를 로컬에 소유 → 네트워크 장애에도 작업
- PR 기반 코드리뷰 → 품질 보증 + 지식 공유 동시
- 2025년 기준 **93.87%** 개발자 사용

### Orbit에 통합 시 희석 위험
- **🟢 없음** — Git은 인프라 계층, AI 앱 계층과 충돌 없음
- **오히려 강화**: AI가 변경사항 감지 → Orbit 맵에 자동 업데이트

---

## 3. Claude — "긴 컨텍스트 + 코드 품질"

### 왜 신봉하는가
- 1M 토큰 컨텍스트 → 전체 프로젝트 한 번에 분석
- 코드 일관성: ChatGPT는 "3/10~9/10 변동", Claude는 "7/10 일관"
- 안전성: 보안/규제 코드에서 신중한 판단

### Orbit에 통합 시 희석 위험
- **🟡 중간** — 긴 컨텍스트는 기반 모델 성능이라 희석 어려움
- 안전성 철학이 "너무 보수적"이라 느끼면 다른 모델 선택
- **Orbit 기회**: Claude 작업이 맵에서 가장 큰 노드 → 영향력 시각화

---

## 4. Perplexity — "실시간 웹 검색 리서치"

### 왜 신봉하는가
- 2025년 최신 정보 실시간 검색 (ChatGPT 지식 컷오프 대비)
- 출처 인용 → 환각 방지
- "리서치 계층"으로 Claude/GPT와 분업

### Orbit에 통합 시 희석 위험
- **🟠 중간-높음** — 웹 검색이 다른 AI에 내장되면 독립 존재 의미 감소
- **Orbit 기회**: Perplexity 검색 결과 노드가 다른 작업 노드와 연결 → "정보 출처 추적"

---

## 5. ChatGPT/GPT-4o — "범용 AI 표준"

### 왜 신봉하는가
- 멀티모달: 텍스트+이미지+오디오 단일 모델
- Custom GPTs + 플러그인 에코시스템
- Microsoft/GitHub/Azure 생태계 깊은 통합

### Orbit에 통합 시 희석 위험
- **🟡 중간** — 이미 "플랫폼 전략"이라 통합에 유리
- **Orbit 기회**: GPT 작업이 n8n, Cursor 작업과 연결 → 전체 흐름 가시화

---

## 6. Gemini — "Google 생태계 통합"

### 왜 신봉하는가
- Google Workspace + Cloud와 자연스러운 연계
- 코드 실행 샌드박스 (즉시 실행 → 결과 확인)
- Gemini 3: 이미지를 능동적 분석 도구로 활용

### Orbit에 통합 시 희석 위험
- **🟠 중간-높음** — Google 생태계 외부 팀에겐 "선택지 중 하나"로 전락
- **Orbit 기회**: Google 이벤트 → Orbit 맵 연동으로 독립성 확보

## Orbit 통합 시 장점이 줄어드는 요소 vs 강화되는 요소

```
도구          희석되는 것                    Orbit에서 강화되는 것
────────────────────────────────────────────────────────────────
Cursor        IDE 독점성                     팀원 파일 작업 실시간 시각화
Git           (희석 없음)                    커밋 → 노드 자동 생성
Claude        범용성 (다른 모델과 같아짐)    긴 세션 흐름 → 행성 크기로 표현
Perplexity    검색 독점성                    검색 결과 → 지식 노드로 연결
ChatGPT       "기본 AI" 이미지              멀티모달 작업이 맵에서 풍부하게 표현
Gemini        Google 한정 이미지            Cloud 작업 흐름 → 팀 전체 공유
```

## 결론: Orbit이 노려야 할 포지션

각 도구는 **기능**을 제공하지만, Orbit은 **맥락(Context)**을 제공한다.

> "Claude가 뭘 했는지는 알지만, 왜 했는지, 누가 다음에 뭘 할지는 모른다"
> → Orbit이 이 공백을 채운다.

**핵심 차별점:**
1. 도구들이 *무엇을 했는지* → 이미 각 도구 자체가 알려줌
2. 도구들이 *어떻게 연결되는지* → Orbit만 보여줌
3. 팀이 *어디로 향하는지* → Orbit의 줌아웃 뷰

이것이 Orbit이 각 도구의 장점을 **희석하지 않고 증폭**시키는 이유.
